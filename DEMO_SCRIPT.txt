[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at how the architecture works under the hood. 
First, when a user enters a query, it passes through the security_checkpoint node, which scrubs any PII like emails or phone numbers, blocks prompt injections, and logs details to a structured JSON audit log. 
If clean, the request routes to our main orchestrator_node, which executes the orchestrator agent. 
The orchestrator agent coordinates delegation using AgentTools to run our specialized sub-agents: the career_advisor, which analyzes skill sets and certificates, and the study_planner, which outlines study milestones. 
Both agents are connected via stdio transport to our local FastMCP server, which supplies database-backed skills, job demand rates, and custom coding exercises. 
Finally, if a roadmap is requested, the workflow routes to our hitl_node, pausing with a RequestInput prompt to ask the user for their weekly hour commitment.
